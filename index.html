Shortest Path

    The Shortest Path Problem is defined as finding a route in a graph between two vertices where the sum of all the edges along that path is minimized.

Approaches

Dijkstra's Algorithm

    Dijkstra's Algorithm takes a graph and a source vertex and finds the shortest path from the source to every other vertex in the graph. Each vertex must also keep track of it's parent or the vertex that comes before it in the path and it's depth or distance from the source node.

Steps:

    1) Set every node's parent to NULL and depth to Infinity, set the source's depth to 0 since its distance from the source is 0.
    2) Create a minimum priority queue (based on each vertex's depth) and push every vertex onto it.
    3) Pop the first vertex (vertex with smallest depth) and relax all it's adjacent vertices.
    4) Repeat step 3 until the queue is empty.
    5) The shortest path from the source to the destination can be found by starting at the destination vertex and going to it's parent, then to that vertex's parent and so on all the way until you arrive at the source.

What is Relaxing?

    Relaxing is the process of determining if the path to the current vertex to the adjacent vertex is shorter than the current path to the adjacent vertex.
    In other words, say the vertex popped from the queue is represented by "min", the current adjacent vertex being relaxed by "curr", and the distance between the two vertices is represented by "dist". Then
    If min.depth + dist < curr.depth,
    then curr.depth = min.depth + dist
    and curr.parent = min
